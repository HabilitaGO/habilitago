# Contribuindo com o HabilitaGO

Obrigado pelo interesse em contribuir! Este documento descreve os padrões e o processo de contribuição.

---

## Antes de Começar

Este é um repositório **público de apresentação** — o código-fonte completo da plataforma está em repositório privado. As contribuições bem-vindas aqui são:

- Correções e melhorias na documentação
- Sugestões de features via Issues
- Relatórios de bugs na plataforma pública
- Melhorias nos diagramas e docs técnicos

---

## Fluxo de Contribuição

1. **Fork** este repositório
2. Crie uma branch com prefixo adequado (veja abaixo)
3. Faça suas alterações
4. Abra um **Pull Request** com o template preenchido
5. Aguarde revisão

---

## Nomenclatura de Branches

| Prefixo | Uso |
|---------|-----|
| `docs/` | Documentação |
| `fix/` | Correção de conteúdo incorreto |
| `feat/` | Nova seção ou funcionalidade de docs |
| `chore/` | Manutenção, estrutura |

Exemplo: `docs/add-mobile-architecture`

---

## Padrão de Commits

Seguimos [Conventional Commits](https://www.conventionalcommits.org/):

```
<tipo>(escopo opcional): resumo em imperativo

[corpo opcional]
```

Tipos: `feat`, `fix`, `docs`, `chore`, `refactor`

Exemplos:
```
docs(architecture): add database schema diagram
fix(readme): correct API endpoint table
feat(product): add pricing model documentation
```

---

## Pull Requests

Use o template ao abrir um PR. Certifique-se de:

- [ ] Descrever claramente o que mudou e por quê
- [ ] Verificar links e referências
- [ ] Revisar ortografia e formatação

---

## Issues

Use os templates disponíveis:

- **Bug Report** — problema na plataforma ou documentação incorreta
- **Feature Request** — sugestão de funcionalidade ou melhoria

**Vulnerabilidades de segurança:** não use issues públicas. Veja [`SECURITY.md`](./SECURITY.md).

---

## Código de Conduta

Seja respeitoso, construtivo e colaborativo. Não toleramos discriminação ou assédio de qualquer tipo.
